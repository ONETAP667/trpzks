# Лабораторна робота №1  
## Розгортання Web-сервісу з автоматизацією

## 1. Загальна інформація

У межах лабораторної роботи реалізовано та автоматизовано розгортання веб-застосунку **mywebapp** на віртуальній машині Linux.

Архітектура системи:

`client → nginx → mywebapp → PostgreSQL`

Система розгортається на одній віртуальній машині.

---

## 2. Варіант індивідуального завдання

### Вихідне число для розрахунку варіанту
`N = 5`

### Розрахунок варіанту
- `V2 = 2`
- `V3 = 3`
- `V5 = 1`

### Отримане індивідуальне завдання

#### Веб-застосунок
**Simple Inventory** — сервіс обліку обладнання.

Об’єкт інвентарю містить поля:
- `id`
- `name`
- `quantity`
- `created_at`

#### API сервісу
- `GET /items` — список усіх предметів (`id`, `name`)
- `POST /items` — створення нового запису (`name`, `quantity`)
- `GET /items/<id>` — детальна інформація про предмет (`id`, `name`, `quantity`, `created_at`)

#### Конфігурація
Застосунок використовує конфігураційний файл:

`/etc/mywebapp/config.yaml`

#### Порт застосунку
Застосунок працює на:

`127.0.0.1:8080`

#### База даних
Використовується:

`PostgreSQL`

---

## 3. Призначення застосунку

Застосунок **mywebapp** призначений для ведення простого обліку обладнання.  
Він дозволяє:
- переглядати список предметів інвентарю;
- створювати нові записи;
- отримувати детальну інформацію про конкретний предмет.

---

## 4. Функціональні можливості

### Бізнес-ендпоінти
- `GET /items`
- `POST /items`
- `GET /items/<id>`

### Health-check ендпоінти
- `GET /health/alive` — завжди повертає `200 OK`
- `GET /health/ready` — повертає `200 OK`, якщо застосунок може підключитися до БД; інакше повертає `500`

### Підтримка форматів відповіді
Застосунок підтримує:
- `text/html`
- `application/json`

Залежно від заголовка `Accept` API повертає або HTML, або JSON.

### Кореневий ендпоінт
- `GET /` — повертає HTML-сторінку зі списком бізнес-ендпоінтів застосунку
